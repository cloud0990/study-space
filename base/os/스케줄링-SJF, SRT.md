SJF 스케줄링 (Shortest Job First)
```javascript
1. 비선점 방식

2. 준비 큐에서 기다리는 프로세스 중, 실행 시간이 가장 짧다고 예상되는 것을 먼저 디스패치

3. 실행할 프로세스의 CPU 사이클이 미리 주어져야만 적용 가능

장점
    - 일괄처리 환경에서 구현하기 쉬움

단점
    - 실행 예정시간의 길이를 사용자의 추정치에 의존 (실제로는 예상 불가)
    - 준비 큐에 새로 들어온 예상 시간이 짧은 작업이, 긴 작업을 기다리게 됨(시분할, 실시간 운영체제 적합X)
```

<br>

SRT 스케줄링 (Shortest Remaining Time)
```javascript
1. 선점 방식

2. 준비 큐의 프로세스 중, 남은 실행시간이 가장 짧다고 예상되는 것을 먼저 디스패치

3. 준비 큐에 새로 들어온 프로세스의 예상 실행시간이 실행 중 프로세스의 남은 실행시간 보다 짧은 경우 즉시 디스패치

4. CPU 사이클이 미리 주어져야만 적용 가능

단점
    - 실행되는 각 프로세스의 실행시간을 추적하여, 각 프로세스가 서비스를 받은 시간을 기록해야함
    - 선점을 위한 문맥교환이 발생하므로, SJF 스케줄링보다 오버헤드가 더 큼

ex.
실행 중 프로세스가 거의 끝날 때쯤,
예상 실행시간이 매우 짧은 프로세스가 입력 또는 실행 중 프로세스의 처리 작업이 많이 남았는데 그것보다 약간 적은 프로세스가 입력되는 경우
선점에 따른 이득 보다 문맥교환의 오버헤드가 더 클 수도 있음
```
