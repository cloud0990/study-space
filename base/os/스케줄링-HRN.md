HRN 스케줄링 (Highest Response Ratio Next)
```javascript
1. 비선점 방식

2. 준비 큐에서 기다리는 프로세스 중, *응답비율이 가장 큰 것을 먼저 디스패치
   오래 기다릴 수록, 실행시간이 짧을 수록 우선순위가 높아짐    
    *응답비율 = 대기시간/예상실행시간 + 1

3. 실행할 프로세스의 CPU 사이클이 미리 주어져야만 적용 가능

4. SJF 스케줄링의 단점 보완
    - SJF: 예상실행시간이 긴 프로세스가 먼저 들어왔어도, 예상 실행시간이 짧은 프로세스에게 계속 밀림
    - HRN: 예상실행시간이 긴 프로세스라도, 대기 시간이 길어지면 응답비율도 커져서 짧은 프로세스가 들어와도 먼저 실행가능
```
