다단계 피드백 큐 스케줄링
```javascript
1. 선점 방식

2. I/O 중심 프로세스와, 연산 중심 프로세스 특성에 따라, 서로 다른 시간 할당량 부여
    - I/O 중심 프로세스: 시간 할당량보다 짧은 실행이 유지되어 작은 단계 유지
    - 연산 중심 프로세스: 시간 할당량 안에 종료되지 못하여 점점 단계가 커짐

3. 각 단계마다 하나씩의 준비 큐가 존재
    - 단계 k는 단계 k+1에 피드백을 줌, 단계가 커질수록 시간 할당량이 커지는 형태
    - 시간 할당량을 모두 소비하면 다음 단계 준비 큐로 이동 배치
    - 단계가 커질수록 디스패치 빈도가 줄어들 수 있음, 시간 할당량도 커져서 한 번 디스패치 되면 오랜 시간 실행할 수 있음
    - 단계 k만 봤을 때는 RR 스케줄링처럼 동작

4. 하나의 단계에서 디스패치 발생 시, 다른 단계에서는 디스패치 불가

ex.
I/O 이벤트 발생 시, 연산 프로세스는 양보하고 대기상태 전이 후, 준비상태가 될 때는 현재와 동일한 단계로 배치
```
