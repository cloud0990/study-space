// 데이터 모델
데이터, 의미, 구조, 연관성 및 데이터의 조건을 표현하기 위한 도구

// 데이터 모델링
데이터의 의미를 파악하고, 데이터와 관여하는 업무 프로세스를 개념적으로 정의/분석 하는 작업

// 데이터 모델링 단계
1. 사용자 요구사항 분석
    - DB 사용 용도 파악
    - 실제 사용자에게 필요한 데이터의 종류/처리방법 등 요구사항을 수집/분석하여 명세서로 작성
    
    // 사용자 요구사항 분석 과정
    제안요청서:RFP (도출)
        - 구축대상, 프로젝트 목표, 범위를 기준으로 조사범위 결정
        - 업무관계자 인터뷰 및 외부자료 수집/분석
    
    요구사항 명세서 (분석)
        - 도출된 요구사항의 명확/완전/모호성 검증 (불완전 -> 도출 단계 재수행)
        - 요구사항을 분류하여 통합/분리
    
    요구사항 정의서 (기록)
        - 분석된 요구사항을 형식에 맞춰 문서화
        - 요구사항 목록 정리 및 관리자 승인
        - 프로젝트 종료까지 반영 여부 지속적 관리

2. 개념적 데이터 모델링
    - 요구사항의 해석 오류 방지/실세계 데이터를 개념적으로 일반화 시켜, 데이터 구조/타입/속성/관계/제약조건 등을 이끌어내는 과정

3. 논리적 데이터 모델링: 개념 스키마 (전체 데이터 구조 포함)
    - 특정 DBMS의 구현 모델에 맞춰 데이터 표현
    - 데이터 정의 언어로 기술된 개념 스키마 생성

4. 물리적 데이터 모델링: 내부 스키마
    - 데이터베이스 파일의 내부 저장구조/파일구성/인덱스/접근경로 등 결정

// ER 모델 (Entity Relationship Model)
1. 실세계의 속성들로 이루어진 개체(Entity)와 개체 사이의 관계(Relationship)를 정형화시킨 모델
    - 개체-관계 표현의 정확성을 위해, 데이터가 준수해야하는 제약조건 정의 가능한 표현 방법 제공

2. 개념적 모델링 단계에서 사용되는 데이터 모델
    
3. 데이터 구조와 관계를 ERD로 표현

4. 구성 요소
    개체 집합
        - 개체(Entity): 개체를 설명하는 여러 속성으로 구성
        - 개체 집합(Entity Set): 같은 속성을 공유하는 개체 집합
    
    관계 집합
        - 관계(Relationship): 개체와 개체 사이의 연관성
        - 관계 집합: 개체 집합 간의 연결 관계

    속성: 개체를 구체적으로 설명, 속성에 포함될 수 있는 값의 특성에 따라 여러 종류로 구분
        - 단순 속성: 더 이상 작은 구성 요소로 나눌 수 없는 속성 (ex. 성별, 나이)
        - 복합 속성: 더 작은 구성 요소로 나눌 수 있는 속성 (ex. 생년월일 - 년 월 일)
        - 단일값 속성: 한 개체에 대해 단 하나의 값만 갖는 속성 (ex. 이름, 나이)
        - 다중값 속성: 한 개체에 대해 여러 값을 갖는 속성 (ex. 전화번호 - 비상연락망 1, 2, 3)
        - 유도 속성: 다른 속성의 값으로부터 값이 유추될 수 있는 속성 (ex. 생년월일 -> 나이 유추 가능)
        - 저장 속성: 유도 속성을 위해 사용될 수 있는 속성 (ex. 나이 <- 생년월일)
~~~

// 제약조건
모든 릴레이션 인스턴스가 만족해야 하는 조건

// 제약조건 종류
1. 사상수 (mapping cardinality): 관계 집합에 참가한 개체 집합에 대해, 한 개체가 다른 개체와 관계를 맺을 수 있는 수량 명사
    - 일대일(1:1) 관계
    - 일대다(1:N) 관계
        ex. x는 y와 하나의 관계만 맺을 수 있다. y는 x에 대해 다수의 관계를 맺을 수 있다.
    - 다대일(N:1) 관계
        ex. y는 x와 하나의 관계만 맺을 수 있다. x는 y에 대해 다수의 관계를 맺을 수 있다.
    - 다대다(N:N) 관계

    +) 화살표가 없는 쪽에 대해 다 관계 (ex. x <- y : x는 y에 대해 다 관계)

2. 참가 제약조건 (participation constraints)
    - 전체적 참가: 어떤 개체 집합의 모든 개체가 관계 집합에 참여하는 조건 (이중선 표현)
    - 부분적 참가: 어떤 개체 집합의 일부 개체가 관계 집합에 참여하는 조건 (단일선 표현)

3. Key
    - 개체를 고유(유일)하게 구분
    - 관계 집합의 특정 관계를 찾음
